# ⚡ Chat Agent Quick Reference Card

## The 4 Core Rules (Remember These!)

### 1️⃣ SHEET NAME IS ALWAYS FIXED
```
🔴 WRONG:  "Sherif's Cases" or "Mark's Cases"
🟢 RIGHT:  "Chat Agent's Cases"  ← Same name every day
```

### 2️⃣ PRESERVE ORIGINAL "ASSIGNED TO"
```
🔴 WRONG:  Change all previous cases to new supporter name
🟢 RIGHT:  Keep old cases with original names
           New cases get current supporter name
```

### 3️⃣ COUNTER SHOWS ONLY ACTIVE HANDLERS
```
🔴 WRONG:  Mark, Sherif, Chat Agent (3 rows - confusing!)
🟢 RIGHT:  Mark, John (just today's active handlers)
           Sherif NOT shown (he's Chat Agent, not handler)
```

### 4️⃣ EXPLICIT LOGGING
```
🔴 WRONG:  Silent code, unclear what happens
🟢 RIGHT:  RULE: messages in logs explain every step
```

## Real-World Example

### Yesterday
- Chat Agent: **Sherif**
- Handlers: **Sherif, Mark**
- Active: Mark only
- Created sheet: "Chat Agent's Cases" ← FIXED NAME
- 50 cases with "Assigned To" = "Sherif"

### Today
- Chat Agent: **Mark**  
- Handlers: **Mark, John**
- Active: John only
- **Find sheet**: "Chat Agent's Cases" ✅ (FOUND! Same name)
- **Load data**: 50 cases with "Assigned To" = "Sherif" ✅ (PRESERVED!)
- **Pull new**: 30 cases from John
- **Assign**: "Assigned To" = "Mark" (current Chat Agent)
- **Result**: 80 cases total (50 old Sherif + 30 new Mark) ✅

## The Flow in 30 Seconds

```
1. User picks handlers: [Mark, John] + Chat Agent "Mark"
   ↓
2. Active handlers = [John] (Mark removed, he's Chat Agent)
   ↓
3. Assign cases to John
   ↓
4. Chat Agent Final Step:
   - Count John's in_progress: 100
   - Pull from bottom: 100 newest cases
   - Change "Assigned To" = "Mark"
   ↓
5. Load previous "Chat Agent's Cases"
   - Keep original "Assigned To" values
   ↓
6. Merge: 100 new (Mark) + 50 old (Sherif) = 150 total
   ↓
7. Write sheets:
   - "John's Cases": John's remaining handwork
   - "Chat Agent's Cases": 150 mixed cases
   - "Counter": Shows John only (Mark not in counter)
```

### ✅ John's Cases

### ❌ Mark's Cases (NOT CREATED)
- Mark is Chat Agent, not handler
- His cases in "Chat Agent's Cases" only

- Mark NOT shown (Chat Agent)

## Troubleshooting

### Problem: Can't find previous Chat Agent cases
**Check**:
```
Previous file has sheet: "Chat Agent's Cases"? ✓
NOT "Sherif's Cases" or other name
```

### Problem: Counter shows Mark & Sherif (confusing)
**Check**:
```
1. selected_handlers passed to create_counters_sheet? ✓
2. Chat Agent filtered out of handler_list? ✓
3. Logs show: "RULE: Counter shows ONLY today's selected handlers"? ✓
```

### Problem: Previous "Assigned To" changed to new supporter
**Check**:
```
Code preserves prev_not_in_new without modifying "Assigned To"? ✓
Did new code modify previous cases? Should be No
```

## Log Messages You Should See

```
RULE: Chat Agent sheet name is ALWAYS 'Chat Agent's Cases' (fixed, not supporter-dependent)
RULE: New pulled cases have 'Assigned To' = 'Mark' (current supporter)
RULE: Previous cases keep their ORIGINAL 'Assigned To' values (might be different supporter)
RULE: Counter shows ONLY today's selected handlers: [John]
RULE: Excluding Chat Agent 'Mark' from ALL counter tables
```

If you don't see these, something's wrong - check logs!
